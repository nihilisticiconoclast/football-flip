Normal by addition

Statistical Rethinking · §4.1.1

A thousand people line up on the halfway line of a football pitch. Each flips a coin sixteen times — heads, step left; tails, step right — then stands still. Measure everyone's distance from the line and a bell curve appears, every single time. Nothing about the curve is built into a coin; it emerges from the summing.

Why a bell?

Each flip contributes one independent ± nudge. An extreme total needs a nearly unanimous run of one face, which is rare; a middling total near zero can be reached in a huge number of ways, so it dominates. Add enough independent nudges and the histogram of totals always tends towards the same shape — the normal distribution. That is the central limit theorem doing its work, and it is why the Gaussian turns up everywhere in the chapters that follow.

The two step models

Coin ±1. With fixed unit steps your final position is just (rights − lefts). The number of rights is Binomial(n, ½), so the position is a shifted, doubled binomial — and the red curve is the de Moivre–Laplace normal approximation to it, with σ = √n (σ = 4 at n = 16). This is the binomial sitting underneath the picture.

Random −1…1. McElreath's actual code randomises the step length as well: each step is drawn from Uniform(−1, 1). The sum is no longer binomial, yet it converges to the very same bell. Here Var(step) = 1/3, so σ = √(n/3) ≈ 2.31 at n = 16 — same centre, narrower spread, because the average step is shorter.

pos <- replicate( 1000 , sum( runif(16, -1, 1) ) ) dens( pos ) # ≈ Normal( 0 , sqrt(16/3) )

What this is (and isn't)

No parameter is being estimated here and there is nothing Bayesian yet — it is a generative argument for why a Gaussian is a reasonable likelihood, and later a reasonable prior. Toggle the step model, drag the number of flips to watch the spread scale as √n, and re-run as many times as you like: every run hangs a fresh thousand under the same red curve.
